# 提炼检查清单

在把 plan mode 对话写进 `docs/learnt` 之前，至少检查下面这些点：

- 语义边界：是否把相似表象但不同语义的场景拆开了
- 所有权：是否写清订单、成交、claim、bucket 归谁收尾
- 状态归属：是否分清外部冻结、STM 自有在途覆盖、未同步成交
- 快照不足以判断：是否指出账户快照为什么不能直接代表策略状态
- 释放条件：是否写清 release task / claim / bucket 的前置条件
- 回归用例：是否给出至少两个相邻场景和一个反例
- 失败风险：是否写出提前释放、重复接管、状态串桶等后果
- 残留问题：是否标出还需要数据源、日志或实现细节才能彻底收敛的点
